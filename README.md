# 自動搶票 Chrome 擴充功能

專為搶票設計的 Chrome 擴充功能（Manifest V3），在開賣瞬間自動搶票。

---

## 功能特色

- **頁面內建面板**：設定介面直接浮動在頁面上，可拖曳移動、收合，不需開啟工具列選單
- **精準倒數重整**：開賣前 2 秒自動重整頁面，確保 DOM 為最新狀態後立即開搶
- **靜默 fetch 輪詢**：不觸發完整頁面 reload，直接抓 HTML 判斷票券可用性
- **票券優先清單**：設定多組票名關鍵字 + 張數，依序嘗試選票
- **Queue API 模式**：直接送出至 queue API，比 DOM 點擊快約 100–300ms
- **自動填表**：自動勾選同意條款、點擊下一步
- **選座頁自動處理**：進入選座頁後自動關閉提示 modal，嘗試點擊電腦配位
- **指數退避**：收到 429/503 時自動退避（2s → 4s → 8s，最多 30s）
- **大圖攔截**：自動阻止 > 200px 圖片載入，減少頁面資源消耗
- **桌面通知**：搶票成功後觸發系統通知

---

## 安裝方式

1. 下載或 clone 此專案
2. 開啟 `chrome://extensions`
3. 右上角開啟「開發者模式」
4. 點擊「載入未封裝項目」，選擇專案資料夾
5. 擴充功能圖示出現即安裝完成

---

## 使用方式

1. 開啟活動報名頁
2. 確認已手動登入
3. 頁面左上角會出現搶票控制面板
4. 填入設定：
   - **開賣時間**：年月日預設今天，手動選擇時分
   - **票券優先清單**：輸入票名關鍵字（如「普通票」、「紅2D」）與張數，按＋新增
   - **輪詢間隔**：預設 1500ms，可依需求調整
   - **自動化選項**：勾選同意條款、下一步、Queue API
5. 按下底部「啟動搶票」開關即開始，開賣前會自動倒數並在適當時機重整頁面開搶
6. 搶到票後 bot 自動停止，付款由使用者手動完成

---

## 設定說明

| 項目 | 說明 |
|------|------|
| 開賣時間 | 倒數基準，開賣前 2 秒自動重整頁面 |
| 票券優先清單 | 依序比對票名，第一個有庫存的票種會被選取 |
| 輪詢間隔 | fetch 輪詢頻率，建議 1000–2000ms |
| 自動勾選同意條款 | 自動勾選購票頁的服務條款 checkbox |
| 自動點擊下一步 | 自動點擊選位／下一步按鈕 |
| 使用 Queue API | 直接 POST 至 queue API，速度更快（進階） |

## 檔案結構

```
manifest.json              Chrome MV3 宣告
content/
  content.js               核心邏輯（面板 UI、輪詢、填表、選座）
background/
  service-worker.js        chrome.alarms 計時、桌面通知
popup/
  popup.html / popup.js / popup.css   （保留備用）
icons/
  icon128.png
```

---

## 注意事項

- 需先手動登入，插件不負責登入流程
- 若目標網站改版導致選擇器失效，請調整 `content.js` 中的 `findTicketRows()` 與 `trySelectTicket()`
- 搶票行為受伺服器端限制，此工具僅加快操作速度，不保證搶票成功
- 部分活動無電腦配位選項，選座頁需手動選座
